# Blink — پروژه‌ی چشمک‌زن LED

━━━ ■ LED ■ ━━━

یک پروژهٔ ساده و جذاب برای چشمک زدن LED — مناسب برای یادگیری پایه‌های سخت‌افزار و برنامه‌نویسی C++ روی بردهای میکروکنترلر یا اجرای ساده با یک برنامهٔ C++ روی سیستم‌های تعبیه‌شده.

ویژگی‌ها
- کد C++ سبک و قابل فهم
- چند الگوی چشمک‌زن (ثابت، چشمک سریع، چشمک کند، پویانمایی)
- نکات سخت‌افزاری و نقشهٔ سیم‌کشی
- راهنما برای ساخت و اجرا روی Arduino و برای کامپایل با g++ (سیستم‌های تعبیه‌شده)

سخت‌افزار موردنیاز
- یک LED
- یک مقاومت 220Ω — 1kΩ (برای محافظت از LED)
- برد توسعه (مثال: Arduino Uno، ESP32، Raspberry Pi Pico یا هر میکروکنترلر دیگر)
- سیم‌های اتصال و بردبرد (Breadboard)

نقشهٔ سیم‌کشی (نمونه برای Arduino Uno)
- سر بلند LED به یکی از پین‌های دیجیتال (مثلاً D13)
- سر کوتاه LED به مقاومت، و سپس به GND

مثال:

  D13 -----|> (LED) --- [220Ω] --- GND

نکات نرم‌افزاری

رایج‌ترین حالت‌ها:
1. Arduino: اگر کد داخل پوشه‌ای به صورت اسکِچ آردوینو باشد (*.ino)، از Arduino IDE یا PlatformIO استفاده کنید.
2. C++ استاندارد: اگر پروژه با C++ استاندارد نوشته شده و از make/CMake پشتیبانی می‌کند، می‌توانید آن را روی دستگاهی که toolchain مناسب دارد کامپایل کنید.

راهنمای سریع — Arduino
1. Arduino IDE را باز کنید.
2. فایل پروژه را باز کنید یا محتوای `Blink.ino` را درون IDE قرار دهید.
3. پورت و برد را تنظیم کنید (Tools → Board و Port).
4. Upload را بزنید.

راهنمای سریع — C++ (g++)
(برای سیستم‌های تعبیه‌شده که توانایی اجرای باینری لینوکس یا cross-compile دارند)

مثال ساده با g++ روی یک سیستم لینوکسی (اگر کد برای آنهدف نوشته شده):

```bash
g++ -std=c++17 -O2 -o blink main.cpp
./blink
```

الگوها و استفاده‌ها
- DEFAULT (چشمک پایه): روشن/خاموش در فواصل مساوی
- FAST (چشمک سریع): فواصل کوتاه‌تر
- SLOW (چشمک کند): فواصل طولانی‌تر
- PATTERN (الگوهای سفارشی): تعریف ماتریس زمان‌بندی یا توالی روشن/خاموش

پیشنهاد پیاده‌سازی (نمونهٔ منطقی C++)
- یک حلقهٔ اصلی که وضعیت LED را با تابع digitalWrite یا کنترل GPIO تغییر می‌دهد
- استفاده از تابع delay یا تایمر غیرمسدود (برای رفتار بهتر)

مشکلات رایج و رفع آن‌ها
- LED چشمک نمی‌زند:
  - بررسی سیم‌کشی و جهت LED (آنود و کاتود)
  - بررسی مقدار مقاومت
  - مطمئن شوید پین صحیح انتخاب شده و روی برد فعال است
- سرعت چشمک مشکل دارد:
  - مقادیر delay/timing را بررسی کنید

نحوهٔ مشارکت
اگر می‌خواهید به پروژه اضافه کنید:
1. یک issue باز کنید و ایده‌تان را توضیح دهید.
2. یک شاخه (branch) جدید از شاخهٔ پیش‌فرض بسازید و تغییرات را ارسال کنید.
3. Pull request بفرستید و توضیح دهید چه چیزی اضافه شده یا رفع شده.
